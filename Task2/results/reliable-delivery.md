# Механизмы обеспечения надежной доставки

## Гарантии доставки Kafka

- At-Least-Once Delivery: для всех доменных событий
- Exactly-Once Semantics: для финансовых событий

## Идемпотентность

- Ключи идемпотентности: хранение в Redis с TTL
- Уникальные ограничения в БД: предотвращение дубликатов на уровне БД
- Идемпотентные операции: проверка существования перед созданием / обновлением

## Retry механизмы

- Экспоненциальная задержка
- Джиттер - случайная вариация задержки
- Классификация ошибок: повторяемые и неповторяемые

## Dead Letter Queue (DLQ)

- Для каждого топика создаем топик DLQ
- Обработка: периодическая автоматическая и ручная
- Мониторинг: алерты при появлении событий в DLQ

## Transactional Outbox Pattern

- Предметная операция и запись события в Outbox в одной транзакции
- Отдельный процесс публикует события в Kafka и помечает их обработку

## Consumer Offset Management

- Ручное управление: коммит только после успешной обработки
- Обработка частичных сбоев: коммит offset только для успешно обработанных событий в batch

## Replication и Durability

- Настройки репликации Kafka: для критичных топиков 3, для обычных 2
- Синхронные реплики: критичные 2, обычные 1
- Настроки acks при публикации: критичные all, обычные 1

## Обработка дубликатов

- На уровне потребителей: проверка ключа идемпотентности перед обработкой
- На уровне БД: проверка уникального ключа
