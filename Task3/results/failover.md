# Аварийное переключение (failover)

[Диаграмма](C4_Failover_Global.png)

- Global DNS/Geo-LB + Global WAF
- Несколько региональных кластеров (EU, SEA, SA, DR):
  - Regional WAF/Firewall
  - API Gateway и Kubernetes Cluster с доменными сервисами
  - Kafka Cluster и DB Cluster (PostgreSQL/ClickHouse и т.д.)
- Между кластерами:
  - CDC-каналы/репликация БД (асинхронная)
  - Kafka MirrorMaker для репликации событий в DR/EU

Таблица ключевых сценариев отказа и переключения:

| Сценарий отказа                         | Влияние                                      | Обнаружение                              | Действия failover                                                                                             | Цель RTO  | Цель RPO   |
| --------------------------------------- | -------------------------------------------- | ---------------------------------------- | ------------------------------------------------------------------------------------------------------------- | --------- | ---------- |
| Отказ одной AZ внутри региона           | Частичная деградация, но регион остается жив | k8s/DB health-check, алерты              | Авто-failover на реплики в других AZ, перераспределение pod-ов                                                | < 5 мин   | 0 мин      |
| Полный отказ регионального кластера SEA | Пользователи ЮВА теряют доступ               | GeoLB health-check региона               | GeoDNS перекидывает трафик в EU/DR, включается DR БД / Kafka                                                  | 15–20 мин | <= 5 мин   |
| Полный отказ регионального кластера SA  | Пользователи Южной Америки теряют доступ     | GeoLB health-check региона               | Трафик в EU/DR, постепенная синхронизация очереди заказов                                                     | 15–20 мин | <= 5 мин   |
| Отказ кластера Kafka в регионе          | Останавливаются асинхронные процессы         | Kafka Exporter метрики, алерты           | Переключение продюсеров/консьюмеров на резервный кластер (в том же регионе или DR), DLQ и переигровка событий | 5–10 мин  | <= 1–5 мин |
| Отказ primary БД доменного сервиса      | Ошибки записи, деградация API                | DB health-check, рост ошибок 5xx         | Авто-переключение на синхронную реплику в регионе, восстановление старого мастера в фоне                      | < 5 мин   | 0 мин      |
| Отказ глобального DNS/GeoLB             | Недоступность по api.gofuture.com глобально  | Внешний мониторинг, multi-provider check | Ручное переключение DNS на резервного провайдера/статический IP edge, включение региональных fallback-доменов | 30–60 мин | 0 мин      |
