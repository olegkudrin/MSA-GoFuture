# Схема геомаршрутизации пользователей

## Механизмы геомаршрутизации

- Глобальный уровень (edge):

  - Пользовательские приложения (Passenger, Driver, Corporate Web) обращаются к глобальному доменному имени api.gofuture.com.
  - Перед целевой инфраструктурой располагаются:
    - Anycast CDN/Edge + WAF (защита от L7-атак, фильтрация трафика, rate limiting);
    - GeoDNS / географический роутинг (аналог Route53 Geo, Yandex DNS, Cloud DNS), который по IP/Geo-IP выбирает ближайший здоровый регион (SEA, SA, EU).

- Региональный уровень:

  - Внутри каждого региона трафик идет:
    - WAF/Firewall региона -> Regional Load Balancer -> API Gateway -> Kubernetes (доменные микросервисы).
  - Привязка пользователя к региону:
    - по умолчанию - ближайший регион по Geo-IP;
    - опционально сохраняется привязка к региону в профиле пользователя/партнера (на случай путешествий и корпоративных договоров);
    - для роуминга можно разрешить переезд между регионами, но с синхронизацией ключевых данных (аккаунт, баланс, истории платежей) через CDC.

- Здоровье регионов и fallback:

  - GeoDNS/глобальный LB периодически проверяет health-check региональных API Gateway.
  - При недоступности региона (health-check failed) пользователи автоматически маршрутизируются в ближайший доступный регион.

## Обоснование механизмов безопасности

[Диаграмма](C2_Security_Global.png) отражает многоуровневую защиту периметра:

- Global WAF (Web Application Firewall):

  - Назначение: фильтрация L7-атак (SQL injection, XSS, CSRF), защита от DDoS, rate limiting на глобальном уровне.
  - Обоснование: единая точка входа api.gofuture.com требует централизованной защиты до распределения трафика по регионам. Снижает нагрузку на региональные WAF и обеспечивает единые политики безопасности.
  - Технологии: Cloud WAF (Yandex Cloud / AWS WAF / Cloudflare), правила на основе сигнатур и поведенческого анализа.

- Global DNS / Geo-LB:

  - Назначение: географическая маршрутизация трафика к ближайшему здоровому региону, распределение нагрузки.
  - Обоснование: минимизация задержек и обеспечение отказоустойчивости на уровне DNS (автоматическое переключение при отказе региона).
  - Технологии: GeoDNS (Route53, Yandex DNS, Cloud DNS), health-check регионов.

- Regional WAF/Firewall:

  - Назначение: дополнительная защита на уровне региона, фильтрация трафика перед API Gateway, изоляция регионов друг от друга.
  - Обоснование: защита от атак, специфичных для региона, соблюдение локальных требований безопасности, снижение риска распространения инцидентов между регионами.
  - Технологии: региональный WAF + Security Groups (firewall rules на уровне сети), интеграция с Identity Service для проверки аутентификации.

- Архитектура защиты:
  - Многоуровневая защита: Global WAF -> Regional WAF -> API Gateway -> Kubernetes (Network Policies).
  - Defense in depth: даже при обходе одного уровня остальные обеспечивают защиту.
  - Соответствие NFR: шифрование TLS, аудит доступа, соответствие требованиям безопасности.
