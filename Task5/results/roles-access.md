# Роли и уровни доступа в мультитенантной платформе GoFuture

## Основные принципы

- Жесткая изоляция данных по tenantId: ни один пользователь партнера не видит данные других партнеров.
- RBAC поверх tenantId: права определяются как (роль _ область действия _ tenantId).
- SSO через IAM: все пользователи аутентифицируются через централизованную IAM с возможной федерацией с IdP партнера.

## Таблица ролей

| Роль                    | Кто это                         | Область действия           | Доступ к данным                                                                         |
| ----------------------- | ------------------------------- | -------------------------- | --------------------------------------------------------------------------------------- |
| GF_PLATFORM_ADMIN       | Админ платформы GoFuture        | Все тенанты (global)       | Полный доступ к конфигурации тенантов, мониторингу, без чтения "сырых" ПДн по умолчанию |
| GF_SUPPORT_ENGINEER     | Поддержка GoFuture              | Все тенанты (ограниченный) | Техническая диагностика, просмотр метрик и логов с маскировкой ПДн                      |
| TENANT_ADMIN            | Админ партнера                  | Свой tenant                | Управление пользователями, настройками, лимитами и фичами только своего тенанта         |
| TENANT_OPERATOR         | Операционный сотрудник партнера | Свой tenant                | Управление поездками, водителями, тарифами в рамках своего тенанта                      |
| TENANT_FINANCE_MANAGER  | Финансовый менеджер партнера    | Свой tenant                | Финансовые отчеты, выплаты, доступ к агрегированным финансовым данным                   |
| TENANT_ANALYST          | Аналитик партнера               | Свой tenant                | Доступ к BI-дашбордам и агрегациям только по своему тенанту                             |
| TENANT_READONLY_AUDITOR | Аудитор/комплаенс партнера      | Свой tenant                | Read-only доступ к логам и ключевым операциям без права изменения                       |
| DRIVER                  | Водитель                        | Свой tenant                | Доступ только к собственным поездкам, профилю и выплатам                                |
| PASSENGER               | Пассажир                        | Свой tenant                | Доступ только к своим поездкам и платежам                                               |

## Уровни доступа

- Конфигурация платформы: только GF_PLATFORM_ADMIN.
- Создание/блокировка тенантов: GF_PLATFORM_ADMIN через Tenant Management Service.
- Управление пользователями тенанта: TENANT_ADMIN внутри своего tenantId (через IAM).
- Финансы и отчеты по партнеру: TENANT_FINANCE_MANAGER, TENANT_ADMIN.
- Технический мониторинг и логи:
  - GoFuture: GF_PLATFORM_ADMIN, GF_SUPPORT_ENGINEER (все tenantId, с маскировкой ПДн).
  - Партнер: TENANT_ADMIN, TENANT_ANALYST, TENANT_READONLY_AUDITOR - только свои tenantId.

## Связь с IAM и SSO

- Все роли маппятся на группы/claims в IAM (Keycloak/Auth0).
- При SSO через корпоративный IdP партнера внешние группы маппятся на роли TENANT\_\*.
- Каждому токену назначается tenantId, по которому ограничивается доступ на уровне API Gateway и доменных сервисов.
