# Архитектура микросервисной платформы GoFuture

## Обзор проекта

Проект представляет собой архитектурное решение для миграции монолитной платформы агрегации такси "GoFuture" на микросервисную архитектуру с поддержкой обработки 500 тысяч конкурентных поездок, глобального масштабирования в нескольких регионах и обеспечения доступности 99,95%.

## Структура решения

### 1. Декомпозиция на домены

Цель: Поэтапный план миграции монолита к микросервисам с минимизацией простоя.

Результаты:

- [Нефункциональные требования](Task1/results/nfr.md) - набор требований по производительности, надежности, безопасности, наблюдаемости
- [Приоритеты выделения сервисов](Task1/results/service-priority.md) - очередность миграции девяти доменных сервисов
- [План миграции](Task1/results/migration-plan.md) - поквартальный план на год с механизмами обратной совместимости
- [Диаграмма](Task1/results/C2_Fundamental.png) - целевая архитектура микросервисов

Ключевые решения: Паттерн Strangler Fig, Database per Service, поэтапная миграция с двойной записью и CDC.

### 2: Event-Driven архитектура

Цель: Проектирование событийной платформы для обработки событий в реальном времени.

Результаты:

- [Схема событий и топиков Kafka](Task2/results/event-schema.md) - доменные события, партиционирование, Saga (Orchestration)
- [Мониторинг](Task2/results/monitoring.md) - инструменты (Prometheus, Grafana, Loki, Jaeger, Schema Registry) и обоснование
- [Метрики](Task2/results/metrics.md) - бизнес-метрики для всех доменов
- [Надежная доставка](Task2/results/reliable-delivery.md) - механизмы гарантий доставки, идемпотентность, DLQ
- [Диаграмма 1](Task2/results/C2_EventDriven.png) - архитектура событийной платформы
- [Диаграмма 2](Task2/results/C2_Infrastructure.png) - архитектура инфраструктуры

Ключевые решения: Apache Kafka с региональным партиционированием, Apache Flink для stream processing, Saga Pattern для распределенных транзакций.

### 3: Глобальное развертывание

Цель: Обеспечение высокой доступности и низкой задержки через развертывание в нескольких географических регионах.

Результаты:

- [Выбор регионов](Task3/results/regions.md) - обоснование EU, SEA, SA, DR
- [Репликация данных](Task3/results/replication.md) - стратегия репликации БД и Kafka между регионами
- [Геомаршрутизация](Task3/results/geo-routing.md) - механизм маршрутизации пользователей к ближайшему региону
- [Failover](Task3/results/failover.md) - план аварийного переключения с таблицей сценариев отказа
- [Комплаенс](Task3/results/compliance.md) - стратегия соответствия локальным регуляторным требованиям
- [Диаграмма 1](Task3/results/C2_Security_Global.png) - архитектура безопасности
- [Диаграмма 2](Task3/results/C4_Failover_Global.png) - архитектура failover

Ключевые решения: Multi-region deployment, асинхронная кросс-регионная репликация, GeoDNS, многоуровневая защита (Global/Regional WAF).

### 4: Data Pipeline для ML и BI

Цель: Единая платформа для сбора, обработки и анализа данных для ML-моделей и BI.

Результаты:

- [Диаграмма](Task4/results/C4_DataPipeline.png) - пайплайн данных из микросервисов в ML-платформу и BI
- [Качество данных](Task4/results/data-quality.md) - механизмы валидации, мониторинга и обеспечения качества данных

Ключевые решения: Kafka Connect для CDC, Stream Processing для агрегаций, Feature Store для ML, ClickHouse для аналитики.

### 5: Мультитенантная платформа

Цель: Дизайн мультитенантной платформы с изоляцией данных, IAM и автоматизированным онбордингом партнеров.

Результаты:

- [Роли и доступ](Task5/results/roles-access.md) - таблица ролей и уровней доступа к данным
- [Диаграмма 1](Task5/results/C2_Multitenant_IAM.png) - архитектура IAM
- [Диаграмма 2](Task5/results/C3_Onboarding_Flow.png) - процесс онбординга

Ключевые решения: Логическая изоляция данных по tenantId, централизованный IAM с SSO, RBAC с привязкой к tenantId.
